# 算法复杂度

先比较两种不同的算法, 在查找数组中(有序数组)给定元素的时间复杂度

方式一: 顺序查找

  • 这种算法从头到尾遍历整个数组, 依次比较每个元素和给定元素的值
  • 如果找到相等的元素, 就会返回下标, 如果遍历整个数组都没找到, 就返回 -1
// 顺序查找
function sequentSearch(arr: number[], num: number): number  {
  for (let i = 0; i < arr.length; i++) {
    const item = arr[i]
    if (item === num) {
      return i
    }
  }
  return -1
}

方式二: 二分查找

  • 这种算法假设数组是有序的, 每次选择数组中间的元素与给定元素进行比较
  • 如果相等, 就返回其下标, 如果给定元素比中间元素小, 则在数组的左半部分进行查找
  • 如果给定元素比中间元素大, 则在数组的右半部分进行查找
  • 这样每次查找都会让查找的返回减半, 直到找到相等的元素或者查找的范围为空
// 二分查找
function binarySearch(arr:number[] , num: number):number {
  // 定义左边的索引
  let left = 0
  // 定义右边的索引
  let right = arr.length - 1

  while (left <= right) {
    // 中间的索引
    let mid = Math.floor((right + left) / 2)
    let midNum = arr[mid]
    if (midNum === num) {
      return mid
    } else if (midNum < num) {
      left = mid + 1
    } else {
      right = right - 1
    } 
  }
  return -1
}

# 空间复杂度

空间复杂度指的是程序运行过程中,所需要的外的存储空间

  • 空间复杂度可以用 大O 表示法来表示
  • 空间复杂度的计算方法与时间复杂度类似,通常需要分析程序中需要额外分配的内存空间, 如数组, 变量, 对象, 递归调用等

# 数组和链表的复杂度对比

使用 大O表示法 来对比数组和链表的时间复杂度

数组是一种连续的存储结构, 通过下标可以直接访问数组中的任意元素

  • 时间复杂度: 对于数组, 随机访问的时间复杂度是 O(1) , 插入和删除的时间复杂度是 O(n)
  • 空间复杂度: 数组需要连续的存储空间, 空间复杂度是 O(n)

链表是一种链式存储结构, 通过指针连接起来的节点组成, 访问俩表中元素需要从头节点开始遍历

  • 时间复杂度 : 对于链表, 随机访问的时间复杂度是 O(n), 插入和删除操作时间复杂度是 O(1)
  • 空间复杂度 : 链表需要为每个节点分配存储空间, 空间复杂度为 O(n)

# 哈希表 HashTable

哈希表通常是基于数组来实现的,但是相对于数组, 它有很多的优势

  • 它可以提供非常快速的 插入-删除-查找 操作
  • 无论是多少数据, 插入和删除值接近于常量的时间, : O(1)的时间复杂度, 实际上, 只需要几个机器指令才可以完成
  • 哈希表的速度比树还要快, 基本可以瞬间查找到想要的元素
  • 哈希表相对于树来说编码要容易很多

哈希表相对于数组的一些不足

  • 哈希表中的数据是没有顺序的, 所以不能以一种固定的方式(比如从小到大), 来遍历其中的元素(没有特殊处理的情况下)
  • 通常情况下, 哈希表中的 key 值是不允许重复的, 不能放置相同的 key, 用于保存不同的元素

哈希表, 它的结构是数组, 但是它神奇的地方就在于, 对数组下标值的一种变换, 这种变换我们可以使用哈希函数, 通过哈希函数可以获取到 HashCode

# 哈希函数的实现

哈希函数
/**
 * 哈希函数: 将 key 字符串映射出来索引值
 * @param key 转换成 key
 * @param max 数组的长度(最大的数值)
 * @returns 索引值
 */
function hashFunc(key: string, max: number): number {
  let hashCode = 0
  const length = key.length

  for (let i = 0; i < length; i++) {
    // 霍纳法则计算的hashCode
    hashCode = 31 * hashCode + key.charCodeAt(i)
  }
  // 余数求出索引值
  const index = hashCode % max
  return index
}

# 插入 & 修改数据 put

哈希表的插入和修改是同一个函数

  • 因为, 当使用者传入一个 <key, value> 时
  • 如果原来不存在该 key, 那么就是插入操作
  • 如果已经存在该 key, 那么就是修改操作

过程: 稍微复杂一点

// [[[key,value],[key,value],[key,value]], [[key,value], [key,value]], [[key,value]]]
class HashTable<T = any> {
  // 创建一个数组, 用来存放连地址法中的链(数组)
  private storage: [string, T][][] = []
  // 定义数组的长度
  private length: number = 7
  // 记录已经存放元素的个数
  private count: number = 0
  
  private hashFunc(key: string, max: number): number {
    let hashCode = 0
    const length = key.length
    for (let i = 0; i < length; i++) {
      // 霍纳法则计算的hashCode
      hashCode = 31 * hashCode + key.charCodeAt(i)
    }
    // 求出索引值
    const index = hashCode % max
    return index
  }


  // 修改 / 插入 操作
  put(key :string , value : T) {
    // 获取索引
    const index = this.hashFunc(key, this.length)
    // 去除索引值对应未知的数组(也可以叫做"桶")
    let bucket = this.storage[index]
    // 判断 bucket 是否有值 
    if (bucket === undefined) {
      bucket = []
      this.storage[index] = bucket
    }
    // 已经确定了这个位置有一个桶(数组)了, 但是不缺定是否存在对应的key
    let isUpdated = false
    for (let i = 0; i < bucket.length; i++) {
      const tuple = bucket[i]
      const tupleKey = tuple[0]
      if (tupleKey === key) {
        // 更新(修改)的操作
        tuple[1] = value
        isUpdated = true
      }
    }
    // 如果在上面代码中的 value 没有被修改(说明没有对应的键值对), 那么在这里执行添加操作
    if (!isUpdated) {
      bucket.push([key, value])
      this.count++
    }
  }
}

  // 创建一个数组, 用来存放连地址法中的链(数组)

  // 定义数组的长度
  private length: number = 7

      // 霍纳法则计算的hashCode
      hashCode = 31 * hashCode + key.charCodeAt(i)
    }
    // 求出索引值
    const index = hashCode % max
    return index
  }
  // 修改 / 插入 操作
  put(key :string , value : T) {
    ...
  }
  // 获取值
  get(key:string):T | undefined {
    // 获取 索引值
    const index = this.hashFunc(key, this.length)
    // 获取 bucket 桶
    const bucket = this.storage[index]

      const tuple = bucket[i]
      const tupleKey = tuple[0]
      if (key === tupleKey) {
        return tuple[1]
      }
    }
    return undefined
  }
}

# 删除操作 delete

  • 根据对应的 key, 删除对应的 key 和 value
class HashTable<T = any> {
  // 创建一个数组, 用来存放连地址法中的链(数组)
  private storage: [string, T][][] = []
  // 定义数组的长度
  private length: number = 7
  // 记录已经存放元素的个数
  private count: number = 0
  
  private hashFunc(key: string, max: number): number {
    let hashCode = 0
    const length = key.length
    for (let i = 0; i < length; i++) {
      // 霍纳法则计算的hashCode
      hashCode = 31 * hashCode + key.charCodeAt(i)
    }
    // 求出索引值
    const index = hashCode % max
    return index
  }
  // 修改 / 插入 操作
  put(key :string , value : T) {
    ...
  }
  // 获取值
  get(key:string):T | undefined {
    ...
  }
  // 删除数据 delete
  delete(key:string): T | undefined {
    // 根据 key 获取索引值
    const index = this.hashFunc(key, this.length)

    // 获取 bucket 桶
    const bucket = this.storage[index]
    if (!bucket) return undefined

    for (let i = 0; i < bucket.length; i++) {
      const tuple = bucket[i]
      const tupleKey = tuple[0]
      const tupleValue = tuple[1]
      if (key === tupleKey) {
        bucket.splice(i, 1)
        this.count--
        return tupleValue
      }
    }
    return undefined
  }
}

# 自动扩容和缩容的操作

扩容函数:

  • 步骤一: 先将之前的数组保存起来, 因为我们待会儿会将storage = []
  • 步骤二: 之前的属性值需要重置
  • 步骤三: 遍历所有的数据项, 重新插入到哈希表中

在什么时候调用扩容方法呢?

  • 在每次添加完新的数据时(也就是 put 方法), 都进行判断

当然 缩容 方法与之相反

class HashTable<T = any> {
  // 创建一个数组, 用来存放连地址法中的链(数组)
  private storage: [string, T][][] = []
  // 定义数组的长度
  private length: number = 7
  // 记录已经存放元素的个数
  private count: number = 0
  
  private hashFunc(key: string, max: number): number {
    let hashCode = 0
    const length = key.length
    for (let i = 0; i < length; i++) {
      // 霍纳法则计算的hashCode
      hashCode = 31 * hashCode + key.charCodeAt(i)
    }
    // 求出索引值
    const index = hashCode % max
    return index
  }
  // 扩容/缩容
  private resize(newLength) {
    // 设置新的长度
    this.length = newLength
    const oldStorage = this.storage
    // 对数据进行初始化操作
    this.storage = []
    // 获取原来的 bucket , 放入新的数组中
    oldStorage.forEach(bucket => {
      for (let i = 0; i < bucket.length; i++) {
        const tuple = bucket[i]
        this.put(tuple[0], tuple[1])
      }
    })
  }
  // 修改 / 插入 操作
  put(key :string , value : T) {
    // 获取索引
    const index = this.hashFunc(key, this.length)
    // 去除索引值对应未知的数组(也可以叫做"桶")
    let bucket = this.storage[index]
    // 判断 bucket 是否有值 
    if (bucket === undefined) {
      bucket = []
      this.storage[index] = bucket
    }
    // 已经确定了这个位置有一个桶(数组)了, 但是不缺定是否存在对应的key
    let isUpdated = false
    for (let i = 0; i < bucket.length; i++) {
      const tuple = bucket[i]
      const tupleKey = tuple[0]
      if (tupleKey === key) {
        // 更新(修改)的操作
        tuple[1] = value
        isUpdated = true
      }
    }
    // 如果在上面代码中的 value 没有被修改(说明没有对应的键值对), 那么在这里执行添加操作
    if (!isUpdated) {
      bucket.push([key, value])
      this.count++
      // 发现 loadFactor 比例大于 0.75 , 那么就直接扩容
      const loadFactor = this.count / this.length
      if (loadFactor > 0.75) {
        this.resize(this.length * 2)
      }
    }
  }
  // 获取值
  get(key:string):T | undefined {
    ...
  }
  // 删除数据
   delete(key:string): T | undefined {
    // 根据 key 获取索引值
    const index = this.hashFunc(key, this.length)

    // 获取 bucket 桶
    const bucket = this.storage[index]
    if (!bucket) return undefined

    for (let i = 0; i < bucket.length; i++) {
      const tuple = bucket[i]
      const tupleKey = tuple[0]
      const tupleValue = tuple[1]
      if (key === tupleKey) {
        bucket.splice(i, 1)
        this.count--
        // 如果 loadFactor 小于 0.25 , 且最小长度为 7 的情况下, 可以缩容
        const loadFactor = this.count / this.length
        if (loadFactor < 0.25 && this.length > 7) {
          this.resize(Math.floor(this.length / 2))
        }
        return tupleValue
      }
    }
    return undefined
  }
}

# 判断数字是不是质数的封装

容量最好是质数

  • 虽然在链地址法中将容量设置为质数, 没有在开放地址法中重要
  • 但是其实在链地址法中质数作为容量有利于数据的均匀分布, 所以完成下一个步骤

先做一下面试题, 判断一个数是不是质数

质数的特点:

  1. 质数也称为素数
  2. 质数表示大于 1 的自然数中 , 只能被 1 和自己整除的数

  const sqrt = Math.sqrt(num)
  for (let i = 0; i <= sqrt; i++){

  return true
}

# 容量质数获取的代码重构

封装获取新的容量质数的代码

class HashTable<T = any> {
  // 创建一个数组, 用来存放连地址法中的链(数组)
  private storage: [string, T][][] = []
  // 定义数组的长度
  private length: number = 7
  // 记录已经存放元素的个数
  private count: number = 0
  
  private hashFunc(key: string, max: number): number {
    let hashCode = 0
    const length = key.length
    for (let i = 0; i < length; i++) {
      // 霍纳法则计算的hashCode
      hashCode = 31 * hashCode + key.charCodeAt(i)
    }
    // 求出索引值
    const index = hashCode % max
    return index
  }
  // 判断是不是质数
  private isPrime(num: number): boolean {
    // 如果掺入的是 8
    // 其实可以只跟平方根比较
    const sqrt = Math.sqrt(num)
    for (let i = 0; i <= sqrt; i++){
      if (num % i === 0) {
        return false;
      }
    }
    // 2-7 都便利完成后, 依然是没有返回 false
    return true
  }
  // 获取新的质数
  private getNextPrime(num:number):number {
    let newPrime = num
    if (!this.isPrime(newPrime)) {
      newPrime++  // 如果不是质数, 就一直 ++
    }
    return newPrime
  }
  // 扩容/缩容
  private resize(newLength) {
    // 设置新的长度
    let newPrime = this.getNextPrime(newLength)
    if(newPrime < 7) newPrime = 7
    this.length =  newPrime

    const oldStorage = this.storage
    // 对数据进行初始化操作
    this.storage = []
    // 获取原来的 bucket , 放入新的数组中
    oldStorage.forEach(bucket => {
      for (let i = 0; i < bucket.length; i++) {
        const tuple = bucket[i]
        this.put(tuple[0], tuple[1])
      }
    })
  }

  ...
  ...
}
