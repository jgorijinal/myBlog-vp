# ++ 和 -- 的位置

运算符 ++ 和 -- 可以置于变量前，也可以置于变量后

  • 当运算符置于变量后，被称为“后置形式”（postfix form）：counter++。
  • 当运算符置于变量前，被称为“前置形式”（prefix form）：++counter。
  • 两者都做同一件事：将变量 counter 与 1 相加。

他们有什么区别吗？

  • 有，但只有当我们使用 ++/-- 的返回值时才能看到区别；
  • 如果自增/自减的值不会被使用，那么两者形式没有区别；
  • 如果想要对变量进行自增操作，并且 需要立刻使用自增后的值，那么我们需要使用前置形式
  • 前置形式返回一个新的值，但后置返回原来的值；
const current = 5
let result = 100
console.log(result +  current++)  // 105

# 线性结构 - 数组

# 线性结构 (linear list)

线性结构: 是由 n(n >= 0)个数据元素(结点) a[0], a[1], a[2],... a[n-1] 组成的有限序列

  • 数据元素的个数 n 定义为表的长度 = list.length (list.length = 0 (表里没有元素)时成为空表)
  • 将非空的线性表 (n >= 1) 记作:(a[0], a[1], a[2], a[3],..., a[n-1])
  • 数据元素 a[i] 这是个抽象符号, 其具体含义在不同情况下可以不同

需要记住几个常见的线性结构即可

# 数组 (Array) 结构

数组(Array) 结构是一种重要的数据结构

  • 几乎是每种编程语言都会提供的一种原生数据结构(语言自带的)
  • 并且可以借助于数组结构实现其他的数据结构: 比如 栈(Stack), 队列(Queue) , 堆(Heap)

通常数组的内存是连续的, 所以数组在知道下标值的情况下, 访问效率是非常高的

# 栈结构 (Stack)

# 认识栈结构

数组:

  • 数组是一种线性结构, 并且可以在数组的 任意位置, 插入和删除数据
  • 但是有时候为了实现某些功能, 必须对这种 任意性 加以 限制
  • 而 栈和队列 就是比较常见的 受限的线性结构

栈 Stack, 它是一种受限的线性结构, 后进先出LIFO

  • 其限制是只允许在 表的一端 进行插入和删除运算。 这一端被称为 栈顶, 相对的, 把另一端称为 栈底
  • LIFO(last in first out) 表示后进入的元素, 第一个弹出栈空间
  • 像一个栈插入新元素又称为进栈/入栈或压栈, 它是把新元素放到栈顶元素的上面, 使之成为新的栈顶元素
  • 从一个栈删除元素又称为出栈/退栈, 它是把栈顶元素删除掉, 使其相邻的元素成为新的栈顶元素

# 封装 栈

栈常见的操作:

  • push() : 添加一个新元素到栈顶位置
  • pop(): 移除栈顶的元素, 同时返回被移除的元素
  • peek(): 返回栈顶的元素, 不对栈做任何的修改 (这个方法不会移除栈顶的元素, 仅仅会返回它)
  • isEmpty(): 如果栈里没有任何元素就会返回 true, 否则会返回 false
  • size(): 返回栈里的元素个数, 这个方法跟数组的 length 属性类似
// 封装一个栈
class ArrayStack {
  // 定义一个数组 , 用于存储元素
  private data: any[] = []
  // 实现栈中相关的一些方法
  // push() : 讲一个元素踏入到栈中
  push(element: any):void { 
    this.data.push(element)
  }
  // pop() : 将栈顶的元素弹出栈 , 并返回移除的元素
  pop():any {
    return this.data.pop()
  }
  // peek():  看一眼栈顶的元素, 但是不进行任何的操作
  peek():any {
    return this.data[this.data.length -1]
  }
  // isEmpty() :  潘丹栈是否为空
  isEmpty():boolean {
    return this.data.length === 0 ? true : false
  }
  // 返回栈的数据的个数
  size():number {
    return this.data.length
  }
}

// 创建 Stack 的实例
const stack1 = new ArrayStack() 
stack1.push('frank')
stack1.push(123)
stack1.push({ name: 'frank' })
console.log(stack1.pop()) // {name:'frank'}
console.log(stack1.peek()) // 123
console.log(stack1.isEmpty()) // false
console.log(stack1.size()) // 2

# 栈面试题 - 十进制转二进制

单词 : 十进制: decimal , 二进制: binary

把余数存入栈中, 然后依次取出

// 十进制 转为 二进制
function decimalToBinary(decimal :number) : string {
  // 1. 创建一个栈 , 用来存放余数
  const stack = new ArrayStack<number>()
  
  // 2. 使用循环
  // while : 不确定次数, 只知道循环结束条件
  // for : 知道循环的次数
  while (decimal > 0) {
    // 余数 result
    const result = decimal % 2 
    stack.push(result)
    
    decimal = Math.floor(decimal / 2)
  }

  // 3.  所有的余数都在 stack 中 , 所以取出即可
  // 只要栈不为空, 一直循环, 取出元素
  let binary = ''
  while (!stack.isEmpty()) {
    // console.log(stack.pop())
    binary += stack.pop()
  }
  return binary
}
console.log(decimalToBinary(35))

# 栈面试题 : 有效的括号

解体思路是 : 一一对应

  • 循环括号字符串, 如果遇到 (左括号, 栈里存放对应的 ) 右括号, 遇到 {, 栈里存放对应的 } .....
  • 然后其他字符串不等于stack.pop() 直接 return false
function isValid(s: string): boolean { 
  const stack = new ArrayStack<string>()
  for (let i = 0; i < s.length; i++) { 
    const c = s[i]
    switch (c) {  // 把对应的符号 入栈
      case '(':
        stack.push(')')
        break
      case '{':
        stack.push('}')
        break
      case '[':
        stack.push(']')
        break
      default: 
        if(c !== stack.pop()) return false  
    }
  }
  return stack.isEmpty() ? true : false  // '(){}['  这种情况最后剩了 [ , 也是无效的
}

console.log(isValid('(){}[]'))   // true
console.log(isValid('(){}['))   // false
console.log(isValid('({})[]'))  // true

# 队列

# 队列的特性

受限的线性结构:

  • 前面总结了一种 受限的线性结构: 栈结构
  • 并且知道了这种受限的线性结构对于解决某些特定的问题, 会有特别的效果

队列, 特使一种受限的线性表, 先进先出 first in last out

  • 受限之处在于它只允许在队列的 前端(front) 进行删除操作
  • 而在队列的 后端(rear) 进行插入操作

生活中的类似的队列结构:

  1. 电影院, 商场, 厕所排队
  2. 优先排队的人, 优先处理 (买票, 结账, wc)

# 队列的常见方法 - 封装类

队列常见的操作:

  • enqueue(element): 向队列尾部添加一个(或多个)新的项
  • dequeue(): 移除队列的第一(即排在队列最前面的项) , 并返回被移除的元素
  • font/peek() : 返回队列中的第一个元素-----最先被添加, 也将是最先被移除的元素,队列不做任何的变动(不移除元素, 只返回元素信息 ----- 与 stack 栈的peek方法类似)
  • isEmpty(): 如果队列不包含任何元素, 返回 true, 否则 返回 false
  • size(): 返回队列包含的元素个数, 与数组的 length 属性类似
interface IQueue<T> {
  enqueue: (element: T) => void
  dequeue: () => T | undefined
  peek: () => T | undefined
  isEmpty: () => boolean
  size: () => number
}

class ArrayQueue<T> implements IQueue<T> {
  // 1. 定义一个数组
  private data: T[] = []
  
  enqueue(element: T):void {
    this.data.push(element)
  }
  dequeue(): T | undefined {
    return this.data.shift()
  }
  peek(): T | undefined {
    return this.data[0]
  }
  isEmpty():boolean {
    return this.data.length === 0
  }
  size():number {
    return this.data.length
  }
}

function hotPotato(names: string[], num: number){
  if(names.length === 0 ) return 
  // 1. 创建队列
  const queue = new ArrayQueue<string>()
  // 2. 循环数组, 把所有元素入栈
  for (const name of names) {
    queue.enqueue(name)
  }
  // 循环的规则
  while (queue.size() > 1) {
    // 1 或 2 , 不淘汰
    for (let i = 1; i < 3; i++) {
      const name = queue.dequeue()
      if(name) queue.enqueue(name)
    }
    // 3 会淘汰
    queue.dequeue() 
  }
  const index = names.indexOf(queue.dequeue()!)
  return index
}

// 约瑟夫环
function lastRemaining(n: number, m: number) {
  const queue = new ArrayQueue<number>()

  for (let i = 0; i < n; i++) {
    queue.enqueue(i)
  }

  while (queue.size() > 1) {
    for (let i = 1; i < m; i++) {
      const num = queue.dequeue()
      if (num) queue.enqueue(num)
    }
    queue.dequeue() 
  }
  return queue.dequeue()!
}

# 链表结构 (LinkedList)

链表和数组一样, 可用于存储一系列的元素. 但是链表和数组的实现机制完全不同

要存储多个元素, 另外一个选择就是链表

但不同于数组, 链表中的元素在内存中不必是连续的空间

  • 链表的每一个元素由一个存储元素本身的节点和一个指向下一个元素的引用(有写语言称为指针或者引用)组成

相对于数组, 链表有一些优点

  • 内存空间不是必须是连续的
    • 可以充分的利用计算机的内存, 实现灵活的内存动态管理
  • 链表不必在创建时就确定大小, 并且大小可以无限的延伸下去
  • 链表在插入和删除数据时, 时间复杂度可以达到 O(1)
    • 相对数组效率高很多

相对于数组, 链表有一下缺点:

  • 链表访问任何一个位置的元素时, 都需要从头开始访问(无法跳过第一个元素访问任何一个元素)
  • 无法通过下标直接访问元素, 需要从头一个一个访问, 直到找到对应的元素

链表类似于火车, 有一个火车头, 火车头会连接一个节点, 节点上有乘客(类似于数据), 并且这个节点会连接下一个节点, 以此类推

# 尾部添加元素 append()

向链表尾部追加一个数据可能有两种情况:

  • 链表本身为空 : 新添加的数据是唯一的节点
  • 链表不为空 : 需要在其他节点后面追加数据

        current = current.next
      }

    }
    this.size ++
  }

# 指定位置插入元素 insert(value, position)

class Node<T> {
  value: T
  next: Node<T> | null

  constructor(value: T) {
    this.value = value
    this.next = null
  }

class LinkedList<T> {
  private head: Node<T> | null = null
  private size: number = 0
  
  get length() {
    return this.size
  }
  // append() : 向链表尾部追加数据
  append(value: T) {
    ...
  }
  traverse() {
    ...
  }
  insert(value: T, position: number): boolean {
    // 边界处理
    if (position < 0 || position > this.size) return false 
    // 1. 创建 Node 节点
    const newNode = new Node(value)
    // 2. 分情况插入: 头部 / 或者其他
    if (position === 0) {
      // 插入到头部的情况
      newNode.next = this.head
      this.head = newNode
    } else {
      // 插入到其他位置的情况
      let current = this.head
      let previous:Node<T> | null = null
      let index = 0
      while (index < position && current) {
        index++
        previous = current
        current = current.next
      }
      newNode.next = current
      previous!.next = newNode
    }

    this.size++
    return true
  }
}

# 根据位置删除节点 removeAt

移除数据有两种常见的方式

  1. 根据位置移除对应的数据
  2. 根据数据, 找到对应的位置, 再移除数据

移除第一项的数据

  • 移除第一项时, 直接让 head 指向第二个数据就可以了
  • 那么第一项数据没有引用指向, 就在俩表中不在生效, 后面就会被回收掉

移除其他项的数据

  • 移除其他项的数据操作是相同的
  • 首先, 需要通过 while 循环找到正确的位置
  • 扎到正确的位置后, 就可以让上一个项的 next 指向 current 项的 next, 这样中间项就没有引用指向它, 也就不在存在于链表中, 后面会被回收掉

  next: Node<T> | null

    this.value = value
    this.next = null
  }

    return this.size
  }

    ...

  traverse() {
    ...
  }
  insert(value: T, position: number): boolean {
   ...

  // 根据位置移除

    if (position < 0 || position >= this.size) return null

    let current = this.head
    if (position === 0) {
      this.head = this.head!.next
    } else {

        previous = current
        current = current.next
        index++
      }
      previous!.next = current?.next ?? null
    }
    this.size--
    return current?.value ?? null
  }
}

function reverseList(head: ListNode | null): ListNode | null {
  // 什么情况链表不需要处理
  // 1. head 本身为 null 的情况
  if (head === null) return null
  // 2. 只有 head 一个节点
  if (head.next === null) return head
  
  // 使用数组模拟栈结构
  const stack:ListNode[] = [] 
  let current = head
  while (current) {
    stack.push(current)
    current = current.next!
  }

  const newHead: ListNode = stack.pop()!
  let newHeadCurrent = newHead
  while (stack.length) {

    newHeadCurrent = newHeadCurrent.next
  }
  newHeadCurrent.next = null   // 终止了环引用(画图会很清晰)
  return  newHead
}

function reverseList(head: ListNode | null): ListNode | null {
  // 什么情况链表不需要处理
  // 1. head 本身为 null 的情况 或者 只有一个节点, 直接返回 head
  if (head === null || head.next === null) return head
  
  // 2. 反转链表结构
  let newHead: ListNode | null = null
  while (head) {
    const current: ListNode | null = head.next
    head.next = newHead
    newHead = head
    head = current
  } 
  return newHead
}
