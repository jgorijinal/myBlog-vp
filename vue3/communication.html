# 非父子组件通信

除了父子组件通信之外, 还会有非父子组件之间的通信

主**要有两种方法

  • provide / inject
  • Mitt全局事件总线

# provide / inject 依赖注入

# provide

为组件后代供给数据

export default {
  provide: {
    message: 'hello!'
  }
}

如果要提供组件内 data()定义 的数,要用函数的形式 , 但这不会保持响应性

export default {
  data() {
    return {
      message: 'hello!'
    }
  },
  provide() {
    // 使用函数的形式，可以访问到 `this`
    return {
      message: this.message
    }
  }
}

为保证provide和inject的响应性链接, 需要使用computed()属性

<script>
import comp from './composition/comp.vue'
import {computed} from "vue";

export default {
  components: {comp},
  data() {
    return {
      message:'message'
    }
  },
  provide() {
    return {
      length: computed(()=>this.message)
    }
  }
}
</script>

# 全局事件总线mitt库

Vue3 从实例中移除了$on,'$off','$once'方法 , 如果想使用全局事件总线, 要通过第三方的库:

  • Vue3官方有推荐的一些库, 例如 mitt 或 tiny-emitter;
  • 先主要了解一下 mitt库open in new window 的使用;

安装

npm install --save mitt
1

utils/eventBus.js

import mitt from 'mitt'

const emitter  = mitt()

export default emitter

# Mitt 取消监听

在某些情况下 , 可能希望取消掉之前注册的函数监听

import emitter from "../utils/eventBus";

// 定义一个函数
function onFoo(){}
emitter.on('foo',onFoo)   // 监听
emitter.off('foo',onFoo)  // 取消监听
